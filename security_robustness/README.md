# Security and Robustness

This folder maps security, robustness, privacy, and adversarial-risk evidence in AI/ML WiFi sensing.

## Focus areas

| Area | Example topics |
|---|---|
| Adversarial ML | Evasion, perturbation, transfer attacks, robustness testing |
| Physical-Layer Risks | CSI perturbation, preamble/pilot manipulation, interference, spoofing |
| Privacy | Leakage of human activity, identity, or behavior patterns from wireless signals |
| Robustness | Noise, domain shift, room changes, device changes, and deployment variation |
| Defenses | Adversarial training, filtering, anomaly detection, randomized smoothing, robust evaluation |

## Purpose

This folder connects technical security risks to AI/ML WiFi sensing systems and helps identify gaps in current research evaluation.
